# Security Policy

If you discover a security issue, please report it privately to the project maintainers through a non-public channel first.

## Guidelines

- Do not publicly disclose details of real data exposure before maintainers have had a chance to assess the issue.
- If you discover leaked credentials, rotate them immediately.
- A `service_role` key must never be placed in frontend code or a public repository.
- Avoid sharing screenshots, logs, or dumps that contain personal data, tokens, or deployment secrets.

## Scope

Relevant reports include:

- Exposed credentials
- Insecure demo defaults
- Unauthorized data access paths
- Privacy leaks in example data
- RLS or auth bypass risks
